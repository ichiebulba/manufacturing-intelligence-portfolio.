# Project 1: Manufacturing Downtime Intelligence Dashboard

## Business Problem

Production teams need visibility into recurring machine stoppages, downtime trends, and high-impact fault categories affecting manufacturing efficiency.

## Dataset

The dataset contains 3 years of simulated production downtime records inspired by manufacturing downtime monitoring processes.

Key fields include:

* Date
* Shift
* Production Line
* Machine
* Fault Category
* Stoppage Reason
* Downtime Duration
* Seasonal Conditions
* Root Cause
* Planned / Unplanned Downtime

## Analysis Performed

* Analysed downtime by machine, line, shift, and fault category
* Identified recurring stoppage patterns
* Compared year-on-year downtime trends
* Highlighted seasonal downtime increases during summer periods
* Performed Pareto analysis to identify top downtime drivers
* Investigated heat-related faults such as inverter trips, drive faults, CPU overheating, and communication timeouts

## Dashboard Features

* Executive KPI overview
* Downtime trend analysis
* Machine performance comparison
* Shift performance comparison
* Root cause analysis
* Seasonal fault analysis
* Pareto chart
* Engineering recommendation view

## Key Insights

* Summer months showed increased downtime linked to heat-related electrical and communication faults.
* A small number of machines and fault categories contributed to a large percentage of total downtime.
* Recurring issues such as tape snapping, box track stoppages, film tracking faults, jaw jams, inverter trips, and communication failures represented key areas for maintenance focus.

## Business Recommendations

* Improve ventilation and cooling around control cabinets during summer.
* Prioritise preventive maintenance for high-frequency fault machines.
* Monitor recurring electrical and communication faults before peak summer months.
* Use downtime trends to support engineering resource planning.

# Project 2: Line 2 Waste Reduction Analytics Dashboard

## Business Problem

Production waste creates financial loss and affects operational efficiency. Management requires visibility into waste trends, root causes, and cost-saving opportunities.

## Dataset

The dataset contains 1 year of simulated Line 2 waste records based on tortilla chip production waste.

Key fields include:

* Date
* Shift
* Machine
* Product
* Waste Type
* Root Cause
* Wheelie Bin Fraction
* Waste Cost
* Severity

Waste cost is calculated using:

* 1 full wheelie bin of tortilla chip waste = £201

## Analysis Performed

* Analysed waste cost by machine, shift, product, and root cause
* Calculated wheelie-bin equivalent losses
* Identified high-cost waste events
* Tracked waste trends over time
* Highlighted recurring waste drivers
* Compared waste performance between shifts

## Dashboard Features

* Total waste cost
* Total wheelie bins lost
* Waste by machine
* Waste by shift
* Waste by product
* Waste by root cause
* Monthly waste trend
* Severity breakdown

## Key Insights

* Certain waste categories contributed more heavily to financial loss.
* Specific machines showed repeated waste patterns.
* Some shifts experienced higher waste events, suggesting process or setup differences.
* Wheelie-bin cost analysis made production losses easier to communicate to non-technical stakeholders.

## Business Recommendations

* Investigate machines with repeated high-cost waste events.
* Review operator setup and changeover procedures.
* Monitor product spill, seal failure, broken product, and start-up waste trends.
* Use visual reporting to support continuous improvement meetings.
